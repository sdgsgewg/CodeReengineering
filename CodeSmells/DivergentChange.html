Divergent Change


Definisi


Divergent Change adalah sebuah code smell yang terjadi ketika sebuah kelas harus diubah berulang kali karena beberapa alasan, seperti perubahan kebutuhan bisnis, alur logika, atau terdapat fitur baru yang ingin ditambahkan, Hal ini juga dapat terjadi ketika suatu kelas memiliki tanggung jawab yang tidak konsisten atau mencoba melakukan terlalu banyak tugas.


Contoh


Before


Code:

Calculator.java



Alasan:

Class 'Calculator' bertanggung jawab untuk melakukan operasi matematika seperti penambahan, pengurangan, perkalian, dan pembagian. Selain melakukan operasi matematika, class 'Calculator' juga bertanggung jawab untuk mencatat atau logging setiap operasi matematika yang dilakukan. Hal ini akan menghasilkan divergent change karena jika ada perubahan terkait cara logging dilakukan atau perluasan fungsi logging, maka class 'Calculator' harus diubah. Sebaliknya, jika ada perubahan terkait operasi matematika, hal itu juga akan mempengaruhi class 'Calculator'.


After


Code:

Calculator.java

Logger.java



Hasil Perbaikan:

Maka dari itu, kita perlu memisahkan dua tanggung jawab tersebut dengan menggunakan teknik 'Extract Class'. Kita akan membuat class baru bernama 'Logger' yang bertanggung jawab untuk mencatat atau logging setiap operasi matematika yang dilakukan. Kemudian kita dapat menggunakan teknik 'Move Method' untuk memindahkan method 'logOperation()' ke class ini. Sehingga saat ini class 'Calculator' hanya bertanggung jawab terhadap operasi matematika. Perubahan ini membuat class 'Calculator' memiliki fokus yang lebih jelas dan mengurangi dampak divergent change. Begitu juga jika ada perubahan logika pada logging, hanya class 'Logger' yang perlu diubah.
