Shotgun Surgery


Definisi


Shotgun Surgery adalah sebuah code smell yang terjadi ketika perubahan kecil dalam code dapat memicu terjadinya perubahan yang harus dilakukan pada banyak bagian dari sitem. Hal ini dapat terjadi ketika suatu fungsionalitas tersebar di beberapa tempat yang berbeda, sehingga membuat perubahan pada satu fitur memerlukan perubahan pada banyak tempat.


Contoh


Before


Code:

UserManager.java

ProductManager.java



Alasan:

Ketika kita ingin mengubah logika dari log aktivitas, maka kita harus memodifikasi metode 'logActivity()' di class 'UserManager' dan 'ProductManager'. Bayangkan juga jika ada lebih banyak class yang menggunakan metode log aktivitas tersebut, hal ini tentu akan memakan waktu yang lama dan tidak efisien.


After


Code:

UserManager.java

ProductManager.java

Logger.java



Hasil Perbaikan:

Maka dari itu, kita perlu melakukan teknik 'Extract Class' untuk membuat class baru bernama 'Logger' untuk menjalankan fungsionalitas log aktivitas. Kemudian kita perlu menggunakan teknik 'Move Method' untuk memindahkan method 'logActivity()' ke class 'Logger' untuk mencegah duplikasi. Dengan dilakukannya perbaikan ini, setiap class yang nantinya perlu menggunakan method ini hanya perlu menambahkan objek logger ke dalam classnya. Selain itu, jika ada perubahan logika 'logActivity()', kita hanya perlu merubah logika pada satu tempat saja.
