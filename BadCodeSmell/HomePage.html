Let's Learn About Code Smells


Apa itu Code Smells?


Code Smells adalah pola atau karakteristik dalam kode yang menunjukkan masalah desain atau implementasi. Code Smells dapat menyebabkan masalah di masa depan atau membuat kode sulit dipahami, dikelola, atau diperbaiki.


Kategori Code Smells


Code Smells umumnya dibagi menjadi 5 kategori yakni:

  • The Bloaters

    Semua code smells dalam kelompok ini menunjukkan bahwa ada bagian dari kode yang terlalu besar. Contoh code smells dengan tipe ini meliputi Large Class, Long Method, Long Parameter List, Primitive Obsession, dan Data Clumps.

  • Change Preventers

    Semua code smells dalam kelompok ini menyebabkan kita menghadapi masalah tertentu ketika kita ingin menambahkan atau memperbarui fitur tertentu. Contoh code smells dengan tipe ini meliputi Divergent Change, Parallel Inheritance Hierarchies, dan Shotgun Surgery.

  • The Couplers

    Semua code smells dalam kelompok ini menunjukkan bahwa ada ketergantungan yang tinggi antara beberapa class (high coupling). Contoh code smells dengan tipe ini meliputi Feature Envy, Inappropiate Intimacy, Message Chains, dan Middle Man.

  • Object-Orientation Abusers

    Semua code smells dalam kelompok ini menunjukkan bahwa struktur kode tidak mengikuti prinsip Object Oriented Programming (OOP) yang tepat. Contoh code smells dengan tipe ini adalah Switch Statements, Temporary Field, Refused Bequest, dan Alternative Classes with Different Interface.

  • The Dispensables

    Semua code smells dalam kelompok ini menunjukkan bahwa ada beberapa bagian dari kode yang kurang berguna dan dapat dihapus. Contoh code smells dengan tipe ini adalah Comments, Data Class, Lazy Class, Dead Code, Duplicate Code, dan Speculative Generality.
